### MEDIUM

**M1: `lastmod` is static and stale for guide pages**

Guide pages use per-guide `g.lastUpdated` values (`2025-11-01` for both guides), which is correct practice. However, all 44 non-guide URLs use the hardcoded constant `SITE_MODIFIED = '2026-05-02'`, which predates today's 14-city expansion (2026-05-06). The 14 newly promoted city pages will carry a `lastmod` four days in the past on launch day. This is a minor signal issue — Google uses `lastmod` as a crawl-scheduling hint — but it understates recency for the new pages.

**Fix:** Update `SITE_MODIFIED` to `'2026-05-06'` in `app/sitemap.ts` to reflect today's expansion, then keep it updated whenever substantive changes ship.

---

### LOW

**L1: `priority` and `changefreq` are present but ignored by Google**

All URLs carry `priority` and `changeFrequency` values. Google has publicly stated it ignores both fields. Their presence does not cause harm, but they add noise. Acceptable to leave; acceptable to strip. No action required.

**L2: Comment in `sitemap.ts` says "12 location pages" — now 26**

Line 46: `// 12 GSC-validated location pages`. This is a stale comment after today's expansion. It will confuse the next developer reading the file.

**Fix:** Update comment to `// 26 GSC-validated location pages (12 original + 14 added 2026-05-06)`.

**L3: `siteConfig.gaId` is a placeholder**

`data/site.ts` line 8: `"gaId": "G-XXXXXXXXXX"`. This is unrelated to the sitemap but worth flagging: if GA is wired up from this config, it will not fire. Not a sitemap issue.
